Introduce Foreign Method(引入外加函数)

你需要为提供服务类增加一个函数，但你无法修改这个类。

在客户类中建立一个函数，并以第一参数形式会传入一个服务实例

动机：
在使用一个类时，它很好，为你提供了所有需要的服务。而后你又需要一项新的服务，这个类却无法提供。
如果可以修改源码，便可以自行添加一个新函数。如果不能。就得在客户端编码，补足你要的那个函数。


重复代码是软件的万恶之源。重复的代码应该被抽出放进同一个函数中。
如果你以外函数实现一项功能。那就是一个明确的信号 ，这个函数原本应该在提供服务的类中实现。

外加函数终归是权宜之计，如果有可能，应该将这些函数插移动它们的理想家园。
如果代码的所有权原因使你无法做这样的搬移。把外加函数交给服务类的拥有者。请他帮你在服务类中实现这个函数。


做法：
1，在客户类中建立一个函数。用来提供你需要功能。
2，以服务类的实现作为该函数的第一个参数。
3，将该函数注释为：“外加函数”，应在服务类中实现。
